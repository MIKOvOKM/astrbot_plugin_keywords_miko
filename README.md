# astrbot_plugin_keywords_miko


> 一个专为解决 **NapCat 媒体缓存易失** 问题而生的 AstrBot 关键字回复插件。支持将图片、语音、视频、文件甚至**合并转发**作为回复内容，通过底层文件互通实现真正的媒体级持久化。

---

## 🔥 部署必看（Docker 用户请重点阅读）

本插件的持久化核心依赖于 AstrBot 与 NapCat 的**底层文件互相访问**。因为 NapCat 自身重启会清空缓存，如果不做目录映射，所有保存的图片/视频在 Bot 重启后都会变成“已过期”。

**如果你使用 Docker 部署，必须满足以下挂载条件：**
1. AstrBot 容器内，必须挂载 NapCat 的 QQ 数据目录（通常是 `/app/.config/QQ`）。
2. NapCat 容器内，必须挂载 AstrBot 的数据目录（通常是 `/AstrBot/data`）。

## ⚠️ 行为说明与已知限制

为了避免不必要的误解，请在使用前了解以下底层机制：

- **平台限制**：目前仅针对 `NapCat + AstrBot` 组合做了深度适配，其他平台请自行尝试。
- **合并转发的副作用**：由于 NapCat 不会对转发消息内的媒体进行缓存，插件采用“Bot 向自身发送私聊消息”的方式强制触发下载。**如果你同时在手机端登录了 Bot 账号，会看到大量消息弹出，这属于正常触发机制，并非 Bug。**
- **添加时的避坑**：
  - 请勿在添加时大量采用包含媒体文件的转发消息，极易导致获取超时。
  - 尽量避免添加包含 Bot 自身 ID 相关内容的合并转发。
- **引用消息（Reply）降级**：处于半支持状态。保存时必须“主动引用消息回复”，不支持保存“转发消息内部的引用”。且在**触发发送时，因协议限制，无法以真实的引用气泡展示，会降级为普通文本+分隔线形式发送**。
- **其他**：屏蔽来自 Bot 自身的消息触发；其余未提到的消息类型均不支持。

## ✨ 核心特性

- **媒体强持久化**：自动下载媒体文件至本地，基于 MD5 全局去重。引用计数归零后自动触发后台垃圾回收，不浪费一寸磁盘。
- **全类型支持**：文本、图片、语音、视频、文件、QQ表情、合并转发（支持嵌套提取）自由组合。
- **智能发送策略**：单条回复走普通消息，超过阈值自动组装为合并转发，避免刷屏。
- **三级作用域**：独立管控群聊关键字、私聊关键字、全局关键字。
- **高可用容错**：针对 NapCat 偶发的“引用节点丢失”问题内置了底层 MD5 硬提取兜底逻辑；内置单用户锁防止并发写入 SQLite 死锁。

## 🚀 快速开始

1. **安装**：在 AstrBot 管理面板 → 插件市场 搜索 `关键字回复系统` 安装；或手动克隆到 `data/plugins/` 目录下重载。
2. **配置**：进入插件配置，**务必填写 `admin_users` (管理员QQ号)**。
3. **开启群组**：在目标群聊发送 `/开启群关键字`。
4. **添加与触发**：发送 `/添加 你好`，随后发送任意内容，最后发送 `/结束添加`。当群成员发送“你好”时，Bot 会将刚才保存的内容原样回复。


### 2. 必要配置
安装后进入插件配置，**务必填写 `admin_users` (管理员QQ号)**，否则无法使用任何管理命令。

### 3. 开启群组
在目标群聊发送 `/开启群关键字`，该群即被纳入白名单。

### 4. 添加与触发
发送 `/添加 你好`，随后发送任意内容（文字、图片、发个合并转发等），最后发送 `/结束添加` 即可。
当群成员发送“你好”时，Bot 会将刚才保存的内容原样回复。
特别说明：全局关键字，仅在”群聊触发”，仅在”私聊添加”。

## ⚙️ 进阶配置说明

大部分配置保持默认即可，以下为可能需要调整的核心项：

- `forward_threshold`：**合并转发触发阈值**。默认 `3`，即回复内容超过 3 条消息时，自动打包为合并转发发送。设为 `0` 则无论多少条都强制合并转发。
- `max_nested_depth` / `max_forward_msg_count`：**合并转发解析限制**。防止单个超大嵌套转发拖死 Bot，默认分别为 3 层和 99 条。
- `multi_user_adding`：**多用户并行添加**。默认关闭（单用户互斥锁模式），开启后允许多个管理员同时添加不同关键字。
- `session_timeout_seconds`：**添加会话超时**。默认 `300` 秒（5分钟），超时自动取消防占用。
- `cleanup_interval_days`：**垃圾回收天数**。媒体文件失去引用后，保留指定天数再删除，默认 7 天。
