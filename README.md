# astrbot_plugin_keywords_miko

> 💡 **AstrBot 强大的多模态关键字回复插件** 
> 支持文本、图片、视频、文件、表情及合并转发等复杂消息的持久化存储与原样回复。不支持添加引用(reply)消息

---

## ✨ 消息类型支持详情

| 消息类型 | 普通消息 | 转发消息 | 合并转发消息 | 备注 |
| :--- | :---: | :---: | :---: | :--- |
| 📝 **文本** | √ | √ | √ | 无限制 |
| 🖼️ **图片** | √ | √ | √ | 无限制 |
| 🎬 **视频** | √ | √ | √ | 无限制 |
| 📁 **文件** | √ | √ | √ | 文件未过期即可正常识别与发送 |
| 😀 **内置表情** | √ | √ | √ | 无限制 |
| 😀 **商城表情** | √ | √ | √ | 仅存结构，过期或下架自动失效 |
| 🎤 **语音** | √ | × | × | 仅限直接收发，无法提取与合转发送 |
| 📦 **合并转发**| √ | √ | √ | **不支持提取 Bot 自身发送的节点** |

### ⚠️ 异常与边界处理机制
*   **文件超限/超时阻断**：添加时若触发文件大小限制或下载超时，将中止当前添加流程并提示。
*   **失效表情静默跳过**：回复时若遇到已失效的商城表情，会自动跳过该表情，不影响后续内容的发送。
*   **Bot 消息提取限制**：无法提取由 Bot 自身发送的合并转发节点（因底层 API 解析会提示消息已过期）。

## ⚠️ Docker 部署必看（核心依赖）

🔥 **本插件为 Docker 部署方案，其他方案请自行尝试。**

本插件的持久化核心依赖于 **AstrBot 与 NapCat 的底层文件互相访问**。因为 NapCat 自身重启会清空缓存，如果不做目录映射，所有保存的图片/视频在 Bot 重启后都会变成“已过期”。

**如果你使用 Docker 部署，必须满足以下“互相挂载”条件：**

### 🗂️ 挂载路径对照表

| 框架 | 容器内默认路径 | 需要映射的宿主机路径 |
| :--- | :--- | :--- |
| **NapCat** | `/app/.config/QQ` | `/你的宿主机路径/A` |
| **NapCat** | `/app/napcat/config` | `/你的宿主机路径/B` |
| **AstrBot** | `/AstrBot/data` | `/你的宿主机路径/C` |

### 🔗 互相挂载示例

*注意：以下配置中的“宿主机路径”请替换为你实际的绝对路径，NapCat 和 AstrBot 必须指向同一个宿主机文件夹以实现互通。*

**Napcat 的 docker-compose 配置片段：**

    /你的宿主机路径/A:/app/.config/QQ # 默认
    /你的宿主机路径/B:/app/napcat/config # 默认
    /你的宿主机路径/C:/AstrBot/data # ⬅️ 关键：挂载 AstrBot 的数据目录

**Astrbot 的 docker-compose 配置片段：**

    /你的宿主机路径/A:/app/.config/QQ # ⬅️ 关键：挂载 NapCat 的 QQ 数据目录
    /你的宿主机路径/C:/AstrBot/data # 默认

> 💡 **自定义路径说明**：需确认 Docker 容器拥有互相访问和移动文件的权限。未测试 Windows 环境下的路径读取和自定义路径情况。

## 🚀 快速开始

### 1. 安装插件
在 AstrBot 管理面板 → 插件市场 搜索 `关键字回复系统` 安装；或手动克隆到 `data/plugins/` 目录后重载。

### 2. 必要配置
进入插件配置，**务必填写 `admin_users`（管理员 QQ 号）**，否则无法使用任何管理命令。

### 3. 开启群组
在目标群聊发送 `/开启群关键字`，该群即被纳入白名单，随后可正常触发关键字。

### 4. 添加与触发
发送 `/添加 你好`，随后发送任意内容（文字、图片、合并转发等），最后发送 `/结束添加`。
当群成员发送“你好”时，Bot 会将刚才保存的内容原样回复。

> 🌐 **关于全局关键字**：仅在“私聊”中添加，仅在加入了“全局白名单”的“群聊”中触发。当同一关键字在私聊/群聊/全局同时存在时，优先发送全局回复，随后再发送局部回复。

> ⚠️ **重点提示**：群聊内添加时 `/结束添加` 命令无效，必须 `@Bot` 结束添加 / 取消添加。

## ⚙️ 进阶配置说明

在插件配置文件中，除了基本的管理员和白名单设置外，还提供以下进阶调优选项：

| 配置项 | 默认值 | 用途说明 |
| :--- | :--- | :--- |
| `forward_threshold` | `3` | **合并转发触发阈值**。当添加的回复内容条数超过此值时，触发时以合并转发形式发送。设为 `0` 则始终使用合并转发。 |
| `max_nested_depth` | `3` | **合并转发嵌套层数**。限制提取“转发中的转发”的深度，设置过大会导致严重的性能问题。 |
| `max_file_size` | `2GB` | **媒体文件大小限制**。超过此大小的文件将被拒绝保存。 |
| `media_timeout_small` | `10s` | **小文件超时**。图片/语音请求的最长等待时间，超时将放弃，防止 NapCat 卡死。 |
| `media_timeout_large` | `60s` | **大文件超时**。视频/文件请求的最长等待时间，建议根据网络环境调整。 |
| `enable_wal_mode` | `开启` | **数据库 WAL 模式**。可提升并发读写性能。如在某些特殊网络文件系统(NFS)下报错，可尝试关闭。 |
| `cleanup_interval_days` | `7天` | **孤儿文件清理周期**。未被任何关键字引用的废弃媒体文件，将在等待该天数后被自动删除。 |

## 📖 指令速查表

> 💡 **提示**：`/` 为你设置的指令前缀，若更改则 `/` 未必生效，但 `@Bot` 一定有效。词表查看权限开放给普通用户是为了充当“互动菜单”。

| 指令 | 说明 | 权限 / 场景 |
| :--- | :--- | :--- |
| `/开启群关键字` | 将当前群加入白名单 | 管理员 / 群聊 |
| `/关闭群关键字` | 将当前群移出白名单 | 管理员 / 群聊 |
| `/开启群全局关键字` | 当前群可触发全局词 | 管理员 / 群聊 |
| `/关闭群全局关键字` | 当前群不可触发全局词 | 管理员 / 群聊 |
| `/添加 <关键字>` | 添加当前作用域关键字 | 管理员 / 群聊或私聊 |
| `/添加全局 <关键字>` | 添加全局关键字（仅私聊可添加） | 管理员 / 仅私聊 |
| `/结束添加` | 完成并保存当前添加 | 管理员 |
| `/取消添加` | 丢弃当前添加 | 管理员 |
| `/删除 <关键字>` | 删除当前作用域关键字 | 管理员 / 群聊或私聊 |
| `/删除全局 <关键字>` | 删除全局关键字 | 管理员 / 任意 |
| `/关键字列表 [页码]` | 查看当前作用域词表 | 所有人 / 群聊或私聊 |
| `/全局关键字列表 [页码]`| 查看全局词表 | 所有人 / 任意 |
| `/群关键字插件白名单` | 查看群白名单列表 | 所有人 / 任意 |
